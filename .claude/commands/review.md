이 프로젝트의 변경 사항을 리뷰하라.

먼저 다음 문서들을 읽어라:
- `/CLAUDE.md`
- `/docs/ARCHITECTURE.md`
- `/docs/ADR.md`

그런 다음 `git diff HEAD` 또는 변경된 파일들을 확인하고, 아래 체크리스트로 검증하라.

## 체크리스트

### 아키텍처
1. **디렉토리 구조**: ARCHITECTURE.md에 정의된 디렉토리 구조를 따르는가?
2. **에러 처리 패턴**: ARCHITECTURE.md의 ApiError 형식과 HTTP 상태 코드 규칙을 따르는가?
3. **데이터 흐름**: 컴포넌트에서 직접 외부 API를 호출하지 않는가?

### 코드 품질
4. **기술 스택 준수**: ADR에 없는 패키지가 추가되었는가? (있다면 ADR 업데이트 필요)
5. **타입 안전성**: TypeScript `any` 또는 `as unknown as` 가 있는가?
6. **CRITICAL 규칙**: CLAUDE.md의 CRITICAL 항목을 위반하지 않는가?

### 테스트
7. **테스트 존재**: 새로운 비즈니스 로직에 대한 테스트가 작성되어 있는가?
8. **기존 테스트**: 기존 테스트가 모두 통과하는가? (`npm test`)

### 보안
9. **인증 체크**: API Route에서 세션/토큰 검증이 있는가?
10. **입력 검증**: 외부 입력(request body, query params)이 검증되는가?
11. **환경변수**: 시크릿이 코드에 하드코딩되어 있지 않은가?

### 빌드 & 품질
12. **빌드 가능**: `npm run build` 에러 없음
13. **린트**: `npm run lint` 경고 없음

## 출력 형식

### 위반 사항

위반이 있을 때만 이 테이블을 출력한다:

| # | 항목 | 파일 | 구체적 문제 | 수정 방안 |
|---|------|------|-----------|---------|
| 1 | {항목명} | {파일:라인} | {문제 설명} | {수정 방법} |

### 통과 항목

위반 없는 항목을 쉼표로 나열한다 (예: 디렉토리 구조, 기술 스택 준수, 테스트 존재 ...).

### 최종 판정

- **PASS**: 모든 항목 통과 — 머지 가능
- **FAIL**: 위반 N개 — 수정 후 재리뷰 필요
- **WARN**: 필수 위반은 없지만 권장사항 N개 — 선택적 수정
